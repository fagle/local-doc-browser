# 工程代码规范

KomiOS 已经不再是一个简单文件预览页，而是包含文件系统浏览、媒体探测、缩略图缓存、Live Photo、视频转码、移动端相册和登录态的本地 Web 桌面。代码规范的目标不是追求形式统一，而是让后续改动不再依赖临场猜测。

## 注释原则

注释应该解释意图、边界和约束，不重复代码本身。

必须写注释的情况：

- 手势识别、事件仲裁、浏览器兼容等依赖事件顺序的代码。
- 缓存 key、去重锁、队列优先级、LRU 清理等容易被误改的策略。
- 文件系统路径安全、跨 Windows/WSL/Linux 路径转换、权限边界。
- 与外部格式兼容的逻辑，例如 Kodi NFO、Live Photo、HEIC、FFmpeg 参数。
- 存在取舍的地方，例如为了移动端体验延迟单击、为了 NAS 稳定避免批量 stat。

不应该写注释的情况：

- 代码本身已经清楚表达的赋值、条件和简单渲染。
- 过期的 TODO、情绪化说明、只描述“做了什么”的流水账。
- 可以通过命名解决的问题。

## 模块边界

客户端模块按职责拆分：

- `app.js` 只做应用状态编排、DOM 入口绑定、模块组装。
- `client/photo-swipe.js` 负责移动端照片查看、滑动、缩放、预加载手势。
- `client/mobile-album.js` 负责移动端目录和文件切换面板。
- `client/live-photo.js` 负责实况照片和 Motion Photo 检测、播放状态。
- `client/video-player.js` 负责视频播放、转码流、进度保存。
- `client/preview-renderers.js` 负责非复杂业务的预览 HTML。

服务端模块按能力拆分：

- `dev-server.mjs` 只保留路由组装和无法继续下沉的薄协调逻辑。
- `server/index-db.mjs` 管理 SQLite schema、缓存和登录数据落库。
- `server/task-queue.mjs` 管理耗时任务队列、优先级和并发。
- `server/path-utils.mjs` 管理路径转换和路径安全。
- `server/file-types.mjs` 管理文件类型判断。
- `server/cache-utils.mjs` 管理缓存命名和 LRU。

新增功能优先放进已有职责模块；只有当一个文件开始同时承担两个以上方向的复杂规则时，才新增模块。

## 函数长度和复杂度

- 普通函数建议控制在 40 行以内。
- 超过 60 行必须考虑拆分，除非它是清晰的模板渲染函数。
- 一个函数里连续出现三层以上 `if/else` 时，应优先抽出策略函数或状态机。
- 异步流程里不要混杂 DOM 更新、网络请求、缓存写入和错误展示；至少拆成“获取数据”和“渲染状态”两步。

## 事件和手势

移动端事件必须按“手势仲裁”设计，不要在多个模块里同时直接处理同一类手势。

- 单击、双击、长按、滑动、缩放之间必须有明确优先级。
- 触摸事件、合成 click、浏览器 dblclick 不能各自独立改状态。
- 放大状态下，单指移动表示平移图片；未放大时，单指横滑表示切图。
- 全屏、抽屉、返回键拦截都必须通过统一 overlay/history 管理。

## 缓存和后台任务

- 缩略图、预览图、媒体信息必须基于源文件路径、大小、修改时间生成缓存 key。
- 后台生成任务必须做并发去重，避免同一文件同时生成多份预览。
- 读取目录时不要一次性 stat 全部文件；文件大小、缩略图和媒体信息都应该懒加载。
- UI 可视区任务优先级高于预加载任务，当前选中文件优先级最高。
- 缓存必须有容量上限和清理策略。

## 移动端和桌面端

移动端和桌面端不是同一套布局简单缩放。

- 桌面端优先文件管理效率、密集信息、快捷操作。
- 移动端优先相册沉浸、滑动切换、底部/抽屉式导航。
- 移动端照片模式应隐藏非核心操作，例如源码、下载原图等。
- 同一个业务能力可以复用数据模型，但 UI 组件可以分开实现。

## CSS 约束

- 复杂模式必须用明确 body class，例如 `photo-mode`、`video-mode`、`document-mode`。
- PC 和移动端样式要分层，不要在一个选择器里同时兼容所有场景。
- 图片和视频主画布不要使用圆角；缩略图和按钮可以使用圆角。
- 深色和浅色模式都要检查，尤其是半透明按钮、Live 按钮、选中态。
- 避免让控件覆盖主内容，除非它是明确的浮层操作。

## 验证要求

每次改动至少执行相关语法检查：

```bash
node --check app.js
node --check client/photo-swipe.js
```

涉及服务端模块时，也检查对应 `.mjs` 文件：

```bash
node --check dev-server.mjs
```

涉及移动端照片体验时，至少人工验证：

- 单击进入全屏。
- 双击放大和复原。
- 放大后单指平移。
- 未放大时左右滑切图。
- 滑到边界时有提示或循环跳转。
- 切换图片不出现裂图、黑屏、尺寸跳变。

## 提交前检查

- 不提交调试目录、临时截图、录屏和日志。
- `package-lock.json` 跟随依赖变更提交。
- 新增依赖前先确认是否真的减少复杂度，而不是把复杂度换一个地方藏起来。
- 修改共享逻辑时，优先补充注释或文档说明边界。
